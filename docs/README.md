# docs/

How documentation is organized in this project.

| Subfolder | What lives here |
|---|---|
| `product/` | PRD, roadmap — what we're building and in what order |
| `architecture/` | Tech stack, data model, data flow, diagrams — how it works |
| `adr/` | Architecture Decision Records — why we chose what we chose |
| `plans/` | Per-milestone implementation plans — how to build each increment |
| `reference/` | Environment variables, API keys, config reference |
| `runbooks/` | Step-by-step operational procedures |
| `devlog/` | Session journals — what happened, when |
| `tutorials/` | How-to guides for common tasks |
