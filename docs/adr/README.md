# Architecture Decision Records

Documents why significant technical decisions were made.

Format: `NNNN-<slug>.md`. Each ADR has:
- **Title** — short name.
- **Status** — Proposed | Accepted | Deprecated | Superseded by NNNN.
- **Context** — what prompted the decision.
- **Decision** — what we chose.
- **Consequences** — what follows from this choice (good and bad).

ADRs are append-only. To reverse a decision, write a new ADR that supersedes the old one.
