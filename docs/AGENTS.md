# AI Agent conventions for docs/

When reading or writing documentation in this directory:

- PRD and roadmap are the source of truth for what to build. Read them first.
- ADRs are append-only. Never edit a merged ADR. Write a new one that supersedes it.
- Plans are frozen once their GitHub Issues are filed. Don't edit mid-execution.
- Devlog entries are one file per session, named `YYYY-MM-DD-<slug>.md`.
- Reference docs stay current — update them when the system changes.
